Image Optimization

Responsive Images

HTMX pages use the same data-optimized attribute as plain HTML. AbsoluteJS rewrites the images at build time, and HTMX handles dynamic partial swaps on top.

Landscape
1200x800 with responsive sizes. Build rewrites src and adds srcset.
src: /assets/jpg/landscape.jpg
Portrait
600x900 portrait. Width/height preserved for CLS prevention.
src: /assets/jpg/portrait.jpg
Square
800x800 square. Same data-optimized attribute, automatic optimization.
src: /assets/jpg/square.jpg

Fill Mode

For fill mode in HTMX pages, use CSS on the parent and data-optimized on the img.

Unoptimized (no data-optimized)

Without the data-optimized attribute, the image is served as-is. Just a normal <img> tag — no build-time rewriting.

Unoptimized
Original JPG served directly. No data-optimized = no transformation.
unoptimized: true
